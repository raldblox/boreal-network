## Why This Structure

The structure intentionally separates:

- thesis and explanation
- commercial category and market truth
- reference taxonomy
- lifecycle behavior
- event contracts
- API contracts
- logical schema
- verification and governance

This keeps implementation work from redefining the domain accidentally.
